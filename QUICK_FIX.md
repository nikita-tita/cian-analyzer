# Быстрое исправление проблемы с сессиями

## Проблема
Railway использует multiple instances без общего хранилища, поэтому сессии теряются между запросами.

## Решение 1: Добавить Redis (РЕКОМЕНДУЕТСЯ)
Откройте https://railway.com/project/7a53c03f-45e5-4795-a9fb-2badeaf9ee16
1. Нажмите "+ New"
2. Выберите "Database" → "Redis"
3. Готово! Приложение автоматически подхватит REDIS_URL

## Решение 2: Ограничить количество инстансов (временное)
1. Откройте Railway Dashboard
2. Settings → Deploy
3. Replicas: установите "1" вместо автоматического
4. Перезапустите сервис

Это заставит все запросы идти на один инстанс, но не масштабируется.
